Choose an issue tracking system

You need to split your goal into several smaller tasks and track their status. For it, you need an issue tracking system. Create a ticket for each feature or bug and update its status in a timely manner. Choose any system that suits you (or your budget): Jira, GitHub, Trello, etc.

Choose a version control system

Use Git. Make branch for each feature or fix. If there is anyone who is ready to do code review, then make pull requests. Then merge feature or fix into your development branch. Remember to use tags to mark versions and other useful metadata for commits.
